# سامانه مدیریت رساله دکتری

این پروژه یک داشبورد فارسی، راست‌چین، مدرن و واکنش‌گرا برای مدیریت پیشرفت رساله دکتری است.

سایت با GitHub Pages منتشر می‌شود، اطلاعات را از Google Sheets می‌خواند و فایل‌های اصلی را از Google Drive باز می‌کند.

## امکانات سامانه

- داشبورد کلی پیشرفت
- نمایش تعداد کلمات فصل دوم
- نمایش منابع مطالعه‌شده
- نمایش وضعیت هفته جاری
- نمودار هدف و عملکرد تجمعی فصل دوم
- برنامه تفصیلی ۱۴ هفته نخست
- صفحه مرور ادبیات
- صفحه روش‌شناسی
- صفحه میدان پژوهش
- صفحه تحلیل و یافته‌ها
- مدیریت منابع
- نمایش فایل‌های Google Drive
- نمایش موعدهای نزدیک
- صفحه تقویم و نقاط عطف
- خط زمانی سالانه رساله
- تشخیص خودکار موعدهای عقب‌افتاده
- نمایش تاریخ‌ها به‌صورت شمسی
- حالت روشن و تاریک
- طراحی سازگار با موبایل و رایانه

---

# ساختار فنی پروژه

```text

    ↓

    ↓
```

# انتشار شیت‌ها به‌صورت CSV

این کار را برای شیت‌های زیر انجام دهید:

```text
برنامه سالانه
پیگیری ۱۴ هفته
مدیریت منابع
بخش‌ها
فایل‌ها
تقویم
```

برای هر شیت:

1. وارد Google Sheets شوید
2. از منوی `File` وارد `Share` شوید
3. گزینه `Publish to web` را انتخاب کنید
4. نام همان شیت را انتخاب کنید
5. نوع خروجی را روی `Comma-separated values (.csv)` بگذارید
6. روی `Publish` کلیک کنید
7. لینک ایجادشده را کپی کنید

نمونه لینک:

```text
https://docs.google.com/spreadsheets/d/e/2PACX-.../pub?output=csv
```

لینک عادی ویرایش Google Sheets برای اتصال مستقیم داده مناسب نیست.

# ایجاد مخزن GitHub

برای سایت شخصی، نام مخزن بهتر است دقیقاً مطابق نام کاربری GitHub باشد.

اگر نام کاربری شما این باشد:

```text
ali-research
```

نام مخزن:

```text
ali-research.github.io
```

آدرس سایت:

```text
https://ali-research.github.io
```

اگر نام مخزن چیز دیگری باشد، آدرس سایت شامل نام مخزن خواهد بود:

```text
https://USERNAME.github.io/REPOSITORY-NAME/
```

# بارگذاری فایل‌ها در GitHub

در مخزن GitHub:

1. روی `Add file` کلیک کنید
2. گزینه `Upload files` را انتخاب کنید
3. فایل‌های پروژه را بارگذاری کنید
4. روی `Commit changes` کلیک کنید

ساختار درست مخزن:

```text
Repository
├── index.html
├── styles.css
├── app.js
├── config.js
└── README-fa.md
```

فایل `index.html` باید مستقیماً در ریشه مخزن باشد.

فایل‌ها را داخل یک پوشه اضافی قرار ندهید.

---

# فعال‌کردن GitHub Pages

در مخزن GitHub:

1. وارد `Settings` شوید
2. وارد بخش `Pages` شوید
3. در قسمت `Build and deployment` گزینه `Deploy from a branch` را انتخاب کنید
4. Branch را روی `main` قرار دهید
5. Folder را روی `/root` قرار دهید
6. روی `Save` کلیک کنید

پس از انتشار، آدرس سایت در همان صفحه نمایش داده می‌شود.

## پیام تنظیم اولیه لازم است

این پیام یعنی لینک CSV یکی از شیت‌ها در `config.js` وارد نشده است.

نمونه:

```js
weekly: "PASTE_WEEKLY_SHEET_CSV_URL_HERE"
```

باید با لینک واقعی جایگزین شود.

## داده‌ها نمایش داده نمی‌شوند

لینک CSV را در یک تب جدید باز کنید.

لینک صحیح باید متن جدولی یا فایل CSV را نمایش دهد.

اگر صفحه ورود گوگل نمایش داده شد، شیت به‌درستی منتشر نشده است.

---

## سایت تغییرات جدید را نمایش نمی‌دهد

تازه‌سازی کامل انجام دهید.

در ویندوز:

```text
Ctrl + Shift + R
```

در مک:

```text
Command + Shift + R
```

همچنین دکمه تازه‌سازی داخل سایت را بزنید.

---

## تقویم خالی است

موارد زیر را بررسی کنید:

- شیت با نام «تقویم» ساخته شده باشد
- ستون‌های عنوان و تاریخ خالی نباشند
- تاریخ به شکل `YYYY-MM-DD` ثبت شده باشد
- لینک CSV تقویم در `config.js` وارد شده باشد

---

# مدل استفاده روزانه

در طول هفته:

- فایل‌های اصلی را در Google Drive ویرایش کنید
- منابع را در شیت مدیریت منابع ثبت کنید
- تاریخ‌های مهم را در شیت تقویم وارد کنید

در پایان هفته:

- عملکرد واقعی را در شیت پیگیری ۱۴ هفته ثبت کنید
- وضعیت هفته را مشخص کنید
- کلمات جدید و مجموع فصل را وارد کنید
- خروجی انجام‌شده را ثبت کنید

در پایان ماه:

- وضعیت برنامه سالانه را به‌روزرسانی کنید
- درصد پیشرفت بخش‌ها را اصلاح کنید
- نقاط عطف انجام‌شده را در شیت تقویم ثبت کنید

---

# ساختار پیشنهادی نهایی سامانه

```text
داشبورد
برنامه هفتگی
تقویم و نقاط عطف
مرور ادبیات
روش‌شناسی
میدان پژوهش
تحلیل و یافته‌ها
منابع
فایل‌ها
```

این ساختار برای مدیریت رساله فردی، ساده، کم‌هزینه و قابل نگهداری است.
